# Easylife API Contract Changelog

本文件记录前后端接口契约变化，不记录 Flutter 或后端内部实现变化。

## 1.1.0 - 2026-07-01 compatible additions

### Added

- 增加 `POST /v1/companion/reply`，用于桌宠陪伴式多轮回复。该接口只返回水上聊天需要的
  `reply`、可选情绪命名、风险等级和可选服务建议，不保存情绪日记。
- 增加 `POST /v1/emotion-journals/summarize`，仅在用户主动保存情绪日记后，根据完整本轮
  对话整理结构化日记内容；接口本身不写数据库。
- 增加 `POST /v1/memories/extract`，从用户已确认保存的内容中提炼低敏长期记忆候选；
  接口不直接修改长期记忆列表。
- 增加上述三个接口的请求、响应 schema 和固定响应示例。

### Decisions

- 旧 `POST /v1/emotion/analyze` 保持兼容；新陪伴页优先使用独立陪伴回复接口，保存时再调用
  情绪日记整理和长期记忆提炼接口。
- AI Provider 失败时客户端可按本地优先策略回退到本地示例能力；服务端仍通过
  `AI_OUTPUT_INVALID` 和 `AI_PROVIDER_UNAVAILABLE` 保持错误边界清晰。

## 1.1.0 - 2026-06-15

### Added

- 增加中国大陆手机号短信验证码登录，新手机号验证成功后自动注册。
- 验证码有效期固定为 5 分钟，60 秒后可重发，同一 challenge 最多允许 5 次错误。
- 增加手机号、设备和 IP 多维发送频率限制的服务端要求。
- 增加访问令牌刷新和当前设备退出登录接口。
- 增加用户画像、宠物档案、情绪日记和长期记忆的增量同步接口。
- 增加本地 Outbox、幂等 mutation、版本冲突和删除墓碑的数据约定。
- 将长期记忆定义为可独立同步和删除的实体。
- 增加二次短信验证后的账号注销接口和异步删除任务响应。
- 增加认证、同步和账号注销请求/响应示例。

### Decisions

- iOS 首发仅支持中国大陆手机号登录；Apple 登录留作后续可选方式。
- 情绪分析与数据保存保持分离，分析接口不自动保存任何记录。
- 只有用户确认记录后，客户端才将情绪日记和长期记忆写入本地并加入同步队列。
- 断网时本地数据优先，联网后自动执行 push 和 pull。
- 第一阶段不同步饮食与体重数据。

## 1.0.0 - 2026-06-15

### Added

- 冻结 `POST /v1/emotion/analyze` 情绪分析接口。
- 定义 Bearer Token 鉴权和 `X-Request-Id` 请求追踪头。
- 定义最小化的 `context`，排除账号、邮箱、完整生日、地址、体重和饮食限制。
- 定义 iOS、Android 和 Web 客户端元数据。
- 定义主情绪标签、辅助标签、情绪强度、可能原因、建议、伙伴回复和伙伴状态。
- 定义 `400`、`401`、`413`、`422`、`429` 和 `503` 标准错误响应。
- 增加请求、成功响应和错误响应 JSON 示例。

### Contract rules

- V1 可以新增可选字段。
- V1 不得删除字段、改变字段类型或改变既有字段语义。
- 破坏性变更必须进入新的主版本路径，例如 `/v2`。
- OpenAPI 文件是接口结构、字段约束和错误码的唯一事实来源。
- 情绪分析接口保持无状态，不自动创建或保存情绪日记。
